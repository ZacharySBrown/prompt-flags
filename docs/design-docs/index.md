# Design Documents

Catalogue of all design documents for the PromptFlags package.

## Format

Each design doc follows this template:

```markdown
# [Feature Name]

## Problem
What problem are we solving? Why now?

## Context
What exists today? What constraints do we have?

## Proposed Approach
How will we solve it? Include architecture, data flow, key decisions.

## Alternatives Considered
What else did we consider? Why did we reject it?

## Acceptance Criteria
How do we know this is done? Concrete, testable criteria.

## Open Questions
What remains unresolved? Tag the staff engineer for judgment calls.
```
